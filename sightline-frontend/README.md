# Sightline — Frontend

React + Vite + TypeScript + Tailwind v4 UI for the [Sightline](../README.md) SME
credit copilot. Reads a loan file, calls `POST /api/analyze` on the backend, and
renders the decision, ratios, red flags, and LLM-drafted memo — with a
human-in-the-loop confirmation checkpoint per EU AI Act Art. 14.

See [`CONTRACT.md`](./CONTRACT.md) for the API and design contract.

## Develop

```bash
npm install
npm run dev
```

Dev server at http://localhost:5173. HMR is on; edit any file in `src/` and the
page updates without a reload.

Other scripts:

- `npm run build` — typecheck (`tsc -b`) then produce a production bundle in `dist/`.
- `npm run preview` — serve the production build locally.
- `npm run lint` — [oxlint](https://oxc.rs).

## Types are generated from the backend's OpenAPI spec

The API types in `src/api/types.ts` are auto-generated by
[`openapi-typescript`](https://openapi-ts.dev). Never edit that file by hand —
regenerate it when the backend contract changes.

```bash
npm run gen:types
```

`gen:types` currently fetches from `http://localhost:8000/openapi.json`, so the
FastAPI backend must be running locally on port 8000 first:

```bash
# from the repo root (not sightline-frontend/)
python -m uvicorn app.main:app --port 8000
```

> **Why local, not the deployed Railway URL?** The current Codespace's egress
> proxy blocks `*.up.railway.app`, so we can't fetch the production spec from
> inside this dev environment. Committing `openapi.json` alongside the generated
> types keeps the contract reproducible. This will be revisited when we set up
> CI/CD — Vercel builds run outside the proxy and can pull the spec directly
> from Railway.
